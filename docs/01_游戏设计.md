# 01 游戏设计

> 一句话:像素风俯视角 Roguelike 生存游戏——参赛者在海岛上自由探索、结盟、搜刮,广播一响全员被拉进随机致命小游戏,活到最后赢走奖金。
>
> 结构参照元气骑士:选角色 → 下地牢 → 推进随机房间 → 捡随机武器 → 死了带宝石回大厅养成。本作把「地牢层」替换成「死亡游戏轮次」。

## 单局流程(30~40 分钟一局)

1. **报名进岛**:选角色,与几十个 AI 参赛者同场(建议 48 人起步,性能允许再加)
2. **沙盒阶段(5~8 分钟)**:在宿舍、食堂、通风管道间自由活动——搜刮道具、和 NPC 结盟、黑市交易、潜入守卫区偷看下一关情报
3. **游戏阶段**:全员传送进随机小游戏,失败者永久淘汰,幸存人数实时递减、奖金池上涨
4. 重复 2→3 直到决赛,最后小规模混战或 1v1
5. **结算**:奖金带回局外,解锁新角色(不同初始技能)、新武器池、设施升级

## 三大设计支柱

### 随机性(Roguelike 核心)
- 每局小游戏顺序和规则变体随机
- 道具掉落随机
- NPC 性格阵容随机
- 隐藏事件随机

### 空间与路线自由
- 岛上是**固定手工地图**(宿舍、食堂、医务室、通风管道、守卫区),小而密,塞满可交互物和秘密通道
- 赢游戏不是唯一活法:贿赂守卫跳过一关、偷换对手道具、破坏装备,甚至集齐线索触发「逃出海岛」的隐藏结局

### 社交自由(工作量大头)
- NPC 有性格(忠诚/投机/狠毒)和好感度
- 结盟直接影响拔河、弹珠等团队关的生死
- 背叛有短期收益和长期代价

## 关卡:俯视角 2D + 双摇杆操作翻译

| 关卡 | 俯视角玩法翻译 |
| --- | --- |
| 木头人 | 巨型娃娃在屏幕顶端,扫描时全图亮起视线区,移动即被狙击;进阶设计:可破坏的掩体 |
| 抠糖饼 | 摇杆/鼠标沿图形轮廓描线,手抖累积裂纹值,限时完成;道具「针头」可降速增稳 |
| 玻璃桥 | 记忆 + 博弈:前排的人替你探路,可以推人换位,或用道具扫描下一块板 |
| 弹珠 | 与结盟 NPC 的 1v1 物理弹射/猜单双博弈,背叛盟友有剧情代价 |
| 抢椅子(原创) | 音乐停止抢安全区,圈外开始刷元气骑士式弹幕清场 |
| 守卫猎杀(原创) | 程序化迷宫中被守卫追捕,完整的元气骑士式战斗:捡武器反杀 |
| 决赛 | 竞技场混战,双摇杆射击系统火力全开 |
